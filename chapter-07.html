为样式规则添加注释

在样式表中，输入/*开始注释，输入*/结束注释，不能将一个注释嵌套在另一个注释内部
在CSS中添加注释，可以标注样式表的主要区域，或者只是针对某条规则或声明进行解释，有助于他人或自己在以后阅读代码

理想继承

当你为某个元素应用CSS属性时，这些属性不仅会影响该元素，还会影响其下的分支元素，也就是说，这些下层的元素继承了其祖先元素的属性
不是所有的属性都会被继承的，对大多数属性来说，还可以使用inherit值强制进行继承
以下是会被继承的CSS属性

层叠：当规则发生冲突时

如果一个元素包含一个以上的类名，在HTML中应使用空格分隔不同的类名
当多条规则同时定义元素的用一个属性时，CSS用层叠的原则来考虑特殊性、顺序和重要性，从而判断相互冲突的规则中哪个规则应该起作用
1.特殊性
特殊性规则指定选择器的具体程度，选择器越特殊，规则就越强，特殊性最高的是ID，最低的是元素名本身，建议在样式表中多使用类选择器，避免使用ID选择器
2.顺序
晚出现的优先级高，例如，直接应用在HTML元素上的规则（不推荐使用这种做法）比外部样式表或插在HTML文档顶部的出现得更晚（因此优先级更高）
3.重要性
可以声明一条特殊的规则覆盖整个系统中的规则，这条规则的重要程度要比其他所有规则高，也可以在某条声明的末尾加上！important，比如 p { color:orange !important; } (除非特殊情况下，否则不推荐使用这种方法)

属性的值

1.inherit
继承祖先元素的属性，比如article元素设置了一个边框，边框通常不会被继承，因此设置 p { border:inherit; } 可以让article里的段落获得相同的边框样式
2.预定义的值
大多数CSS属性都有一些可供使用的预定义值，要准确地输入这些值，并且不要加上引号
3.长度和百分数
很多CSS属性的值是长度，所有长度都必须包含数字和单位，且它们之间没有空格，有的长度是相对于其他值的，一个em的长度大约与对应元素的字号相等，例如，对元素设置margin-left:2em表示将元素的左外边距设为元素字号的两倍，百分数的工作方式很像em，它们都是相对于其他值的值，百分数通常是相对于父元素的
4.纯数字
只有极少数的CSS属性接受不带单位的数字，其中最常见就是line-height、z-index和opacity
5.URL
CSS属性中的URL无需使用引号包围，这里相对URL应该相对于样式表的位置，而不是相对于HTML文档的位置
6.CSS颜色
目前为止最为常见的定义颜色的方式是使用十六进制格式（通常称为hex）或RGB格式
CSS3引入了另一种指定颜色的方式HSL，以及通过RGBA和HSLA设置alpha透明度的能力（使用十六进制无法指示alpha透明度）

7.RGB
可以通过指定红、绿、蓝的量来构建自己的颜色，可以使用百分数、0~255之间的数字来指定这三种颜色的值，例如，如果创建一种深紫色，可以使用89份红、127份蓝、0份绿，颜色可以写作rgb(89, 0, 127)或者rgb(35%, 0%, 50%)
8.十六进制
将红、绿、蓝的值转化为十六进制，然后在前面加一个#，就表示这个颜色的十六进制，比如rgb(89, 0, 127) = #59007F
如果一个十六进制颜色是由三对重复的数字组成的，如#ff3344，可以缩写为#f34
9.更多CSS3提供的指定颜色的方式：RBGA、HSLA和HSL
RGBA:
RGBA在RGB的基础上加了一个代表alpha透明度的A，可以在红、绿、蓝数值后面加上一个用以指定透明度的0到1之间的小数
alpha设置越接近0，颜色就越透明，如果设为0，就是完全透明，设为1，就是完全不透明
HSL:
HSL代表色相(hue)、饱和度(saturation)和亮度(lightness)，其中色相的取值范围为0~360，0和360在顶部汇合，代表同一种颜色：红色，饱和度和亮度的取值均为百分数，范围为0~100%，HSL语法为：hsl(hue, saturation, lightness)，比如： 红色、 黄色、 绿色、 青色、 蓝色、 紫红色
HSLA:
HSLA是除RBGA以外为颜色设置alpha透明度的另一种方式，语法为：hsla(hue, saturation, lightness, alpha transparency)，比如： 70%的透明度、 25%的透明度
